# AGENTS.md

## 项目定位

本项目用于**快速、准确地管理飞书多维表格中的核心业务数据**。当前核心范围包括：

- 客户信息
- 成交 / 订单
- 用户权益 / 交付记录
- 内容资产
- 特殊伙伴档案

这里的根原则只有两个：

1. **先定位正确的库、表、知识库节点，再执行操作**
2. **关键业务逻辑固定走对应 skill，不要自行发明流程**

## 先读什么

任何实际操作前，先读这些文件：

1. `00_全局配置/飞书基本信息.md`
2. 与任务对应的 skill 文档
3. 相关补充规则文档

本地 skill 文档入口：`00_全局配置/skills/`

补充规则文档：

- `00_全局配置/产品信息和权益.md`
- `00_全局配置/特殊客户档案管理.md`

## 任务分流

遇到任务时，按下面路由，不要混用职责。

### 1. 客户信息

涉及客户**新增、修改、查询**，必须走 `sisi-client`。

适用范围：

- 客户昵称、真实姓名、手机号、微信号等识别信息
- 客户标签、客户类型、归属人、来源渠道
- 客户查重、相似记录确认

禁止事项：

- 不经查重直接新增客户
- 命中多条候选时擅自选择

### 2. 新成交 / 新订单 / 付款确认

涉及客户新成交，必须走 `sisi-deal-and-benefit`。

必须同时处理：

- `成交表`：记录客户、产品、实际金额、订单号、负责人、阶段
- `用户权益明细表`：按产品规则拆分权益

前置要求：

- 先确认客户存在；不存在时先转 `sisi-client`
- 先确认产品存在
- 先读取 `00_全局配置/产品信息和权益.md`

禁止事项：

- 只写成交，不写权益
- 未确认产品规则就猜测权益
- 同一订单号疑似重复时直接再建一条

### 3. 内容资产

涉及内容资产**新增、修改、查询**，必须走 `sisi-content-asset`。

核心关系：

- 内容资产归属于 `内容资产表`
- 每条资产必须定位到正确的 `活动交付记录`
- 子资产要挂到正确的汇总记录 / 父记录下

禁止事项：

- 不确认活动就直接写内容资产
- 凭空在内容资产里伪造活动信息

### 4. 特殊伙伴档案

涉及**重要客户 / 嘉宾 / 合作伙伴**档案，必须走 `sisi-partner-profile`。

该类任务同时可能涉及：

- 飞书知识库子页面创建
- 父页面目录表格更新
- 必要时同步客户名单中的 `档案页` / `客户标签`

补充规则来自：

- `00_全局配置/特殊客户档案管理.md`

禁止事项：

- 未确认类型就建档
- 信息不完整时用猜测补齐
- 已有同名 / 同人档案时重复创建

## 执行原则

### 1. 先定位，再写入

所有写操作前，先确认：

- Base ID 是否正确
- table ID 是否正确
- link 字段目标表是否正确
- 当前任务是否应该转给现有 skill

### 2. skill 固定关键逻辑

本项目中，**AGENTS 负责定位数据源和任务路由，skill 负责固定业务逻辑**。

因此：

- 不要在根流程里重写 skill 的详细业务规则
- 不要绕过 skill 直接改核心业务表
- 只有 skill 未覆盖的辅助性只读检查，才允许临时直接查询

### 3. 所有真实写操作优先使用用户身份

若使用 `lark-cli`：

- 显式带 `--as user`
- 写入前先确认字段类型
- link 字段必须先解析目标 `record_id`

### 4. 准确性优先于速度

遇到以下情况必须停下确认：

- 命中 0 条，但存在疑似相似记录
- 命中多条候选记录
- 产品权益规则不明确
- 档案类型不明确
- 用户提供的信息不足以唯一定位对象

## 你不应该做的事

- 把 `space_id`、wiki token、table ID 混用
- 跳过查重直接新增客户
- 不经确认覆盖已有记录
- 猜测 link 目标的 `record_id`
- 在未知产品规则下擅自拆权益
- 在错误父节点下创建伙伴档案

## 最小工作方式

收到任务后，优先按这个顺序思考：

1. 这是哪一类业务对象？
2. 对应哪张表或哪个知识库节点？
3. 是否必须调用既有 skill？
4. 是否已经具备唯一定位所需的信息？
5. 若没有，先补信息；若有，再执行。
