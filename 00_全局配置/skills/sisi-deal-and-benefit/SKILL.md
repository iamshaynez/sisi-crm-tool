# sisi-deal-and-benefit

## 何时使用

- 任何**新的客户成交**场景（新订单、新付款、赠送确认等）。
- 必须同时维护两张表：
  - **成交表** `tblU9xWQEC9vt5yb`
  - **用户权益明细表** `tbl5PQgyV5vfKia0`
- 本 skill 不负责创建客户，客户操作必须转 `sisi-client`。

## 前置读取（每次执行前必须做）

读取 `00_全局配置/飞书基本信息.md`，确认关键 ID：

- **Base（库）Token**：`IWFEbuZcfalvQus6vkOcJXUjn2d`
- **成交表 ID**：`tblU9xWQEC9vt5yb`
- **用户权益明细表 ID**：`tbl5PQgyV5vfKia0`
- **客户名单表 ID**：`tblvKLGIHObVQ3dV`
- **产品目录表 ID**：`tblljYU2iuLOOb5F`
- **团队成员表 ID**：`tbl7HlE2tDnNCbwC`

所有命令必须显式使用 `--as user`。

## 执行流程（严格顺序）

### 1. 收集成交信息

向用户收集以下核心字段，缺一不可：

| 字段            | 说明                                               | 写入目标              |
| --------------- | -------------------------------------------------- | --------------------- |
| 客户标识        | 客户昵称/真实姓名/手机号/微信号/微博/抖音/小红书等 | 客户名称（link）      |
| 购买产品        | 产品目录中的产品名称                               | 意向产品（link）      |
| 实际成交金额    | 用户实际支付的金额，不是目录价                     | 实际金额（number）    |
| 付款渠道信息    | 微店订单号为主，其他渠道写备注                     | 订单号 / 支付信息备注 |
| 成交客服/负责人 | 负责该成交的团队成员姓名                           | 负责人（link）        |
| 交付日期        | 如有明确交付/服务日期则填写                        | 交付日期（datetime）  |

### 2. 确认客户存在

**必须调用 `sisi-client` skill** 查询/确认客户：

- 如果客户已存在：记录其 `record_id`。
- 如果客户不存在：转 `sisi-client` 新增客户，拿到 `record_id` 后再继续。
- 如果存在多条疑似记录：必须请用户选择具体 `record_id`，禁止擅自决定。

### 3. 确认产品和负责人

#### 查询产品

```bash
lark-cli base +record-search \
  --base-token IWFEbuZcfalvQus6vkOcJXUjn2d \
  --table-id tblljYU2iuLOOb5F \
  --keyword "<产品名称>" \
  --search-field 产品名称 \
  --limit 20 \
  --format json \
  --as user
```

- 命中 **0 条**：停止，询问用户产品名称是否正确。
- 命中 **多条**：列出候选产品的「产品名称 + 价格 + 套餐包含 + record_id」，请用户选择。
- 命中 **1 条**：记录 `record_id`。

#### 查询负责人

```bash
lark-cli base +record-search \
  --base-token IWFEbuZcfalvQus6vkOcJXUjn2d \
  --table-id tbl7HlE2tDnNCbwC \
  --keyword "<负责人姓名>" \
  --search-field 昵称 \
  --search-field 真实姓名 \
  --limit 20 \
  --format json \
  --as user
```

- 命中 **0 条或多条不确定**：请用户确认或提供更多线索。
- 命中 **1 条**：记录 `record_id`。

### 4. 新增成交记录

```bash
lark-cli base +record-upsert \
  --base-token IWFEbuZcfalvQus6vkOcJXUjn2d \
  --table-id tblU9xWQEC9vt5yb \
  --json '<成交记录 JSON>' \
  --as user
```

成交记录 JSON 示例：

```json
{
  "客户名称": [{"id": "<客户 record_id>"}],
  "意向产品": [{"id": "<产品 record_id>"}],
  "实际金额": 4999,
  "阶段": "已付款",
  "负责人": [{"id": "<负责人 record_id>"}],
  "订单号": "WD202607070001",
  "支付信息备注": "微信支付 / 微店",
  "交付日期": "2026-07-07"
}
```

记录返回的 `record_id` 作为成交记录 ID。

### 5. 维护用户权益明细

根据购买的产品，按下方「产品 → 权益映射规则」拆分出应创建的权益记录，每条权益单独创建。

```bash
lark-cli base +record-upsert \
  --base-token IWFEbuZcfalvQus6vkOcJXUjn2d \
  --table-id tbl5PQgyV5vfKia0 \
  --json '<权益记录 JSON>' \
  --as user
```

权益记录 JSON 示例：

```json
{
  "客户名称": [{"id": "<客户 record_id>"}],
  "交付产品": [{"id": "<产品 record_id>"}],
  "来源记录": [{"id": "<成交记录 record_id>"}],
  "交付物": "斯斯1v1深度咨询",
  "状态": "未交付",
  "有效期": "2027-07-07",
  "交付说明": "斯斯90分钟1v1商业咨询 - 单人 2026"
}
```

## 字段写入规范

写入前务必先用 `lark-cli base +field-list` 核对字段类型，再按以下规则构造 `CellValue`：

| 字段类型           | 写入格式                                  | 示例                           |
| ------------------ | ----------------------------------------- | ------------------------------ |
| text / phone / url | 字符串                                    | `"13800138000"`              |
| select（单选）     | 选项名字符串                              | `"已付款"`                   |
| select（多选）     | 选项名数组                                | `["VIP", "业务阶段 1-10"]`   |
| number / currency  | 数字                                      | `4999`                       |
| datetime           | `YYYY-MM-DD` 或 `YYYY-MM-DD HH:mm:ss` | `"2026-07-07"`               |
| link               | `[{"id": "<目标 record_id>"}]`          | `[{"id": "recvkNdMOzezuP"}]` |

### 只读字段（禁止写入）

- 成交表：`线索标题`（formula）
- 用户权益明细表：`交付项名称`（formula）
- 通常只读：`创建日期`

如果返回中出现 `ignored_fields` 或 `READONLY`，立即移除对应字段再重试。

## 安全与错误处理

- **必须先确认客户**：成交记录创建前，必须通过 `sisi-client` 确认客户存在并拿到 `record_id`。
- **禁止重复创建成交**：同一订单号、同一客户 + 同一产品 + 相近时间，需先查询成交表是否已存在。
- **禁止猜测 ID**：link 字段、select 选项必须来自真实命令返回。
- **权益必须拆分完整**：套餐产品必须按规则拆成多条权益，不能只做一条记录。
- **阶段字段**：新成交默认 `已付款`；如为赠送或退款，按实际情况填写。
- **scope / 权限错误**：如果 user 身份报授权不足，先转 `lark-shared` 处理认证，不要直接切换 `--as bot`。
- **常见错误码**：
  - `param baseToken is invalid`：检查是否错把 wiki token 或 space_id 当成 base token。
  - `1254045` 字段名不存在：重新 `+field-list`，使用真实字段名或字段 ID。
  - `ignored_fields` / `READONLY`：移除只读字段后重试。

## 输出要求

- 新增成交后，向用户确认：客户名称、购买产品、实际金额、负责人、订单号、成交记录 `record_id`。
- 创建权益后，列出每条权益的：交付物、数量、状态、有效期、权益记录 `record_id`。
- 如果发现信息缺失或产品无法识别，必须向用户确认后再继续，禁止静默跳过。
